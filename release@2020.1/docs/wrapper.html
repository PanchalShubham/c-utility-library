utility/wrapper.h

[since release@2020.1]

Provides wrapper for primitive types!
Allocates/deallocates memory on heap!
Designed for ease of implementation of containers!
All operations are done in constant time!

The following example creates a linked list of integers using wrappers! 

// create a new empty linked list
LinkedList* list = new_linked_list();

// insert integer 10 into list - but function expects void* so we use wrapper to wrap 10 under void* on heap
linked_list_insert_front(list, wrap_int(10));

// insert integer 20 into list - but function expects void* so we use wrapper to wrap 10 under void* on heap
linked_list_insert_back(list, wrap_int(20));

// get the value at index 0 in linked list but function returns void* so we unwrap it using wrapped_int_value
int value = wrapped_int_value(linked_list_value_at(list, 0));

// get the value at index 0 in linked list but function returns void* so we unwrap it and free memory on heap
int value = unwrap_int(linked_list_delete_at(list, 0));

Function description
void* wrap_int(int value)
  • allocate memory on heap for one integer with given value
  • returns pointer to newly allocated memory on heap if succeeds otherwise returns NULL
  • since release @2020.1
int wrapped_int_value(void* wrapped_int)
  • gets the value contained in wrapped_int but do not delete memeory on heap
  • returns the integer value contained in wrapped_int
  • asserts that wrapped_int is NOT NULL
  • behavior is undefined if other parameter is passed instead of a wrapped_int created using wrap_int call
  • since release @2020.1
int unwrap_int(void* wrapped_int)
  • gets the value contained in wrapped_int and deletes memeory on heap
  • returns the integer value contained in wrapped_int
  • asserts that wrapped_int is NOT NULL
  • behavior is undefined if other parameter is passed instead of a wrapped_int created using wrap_int call
  • since release @2020.1
